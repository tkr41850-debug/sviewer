Implement `computeAllMetrics` in `src/metrics/engine.ts`. The function takes `MetricsInput` (records, metadata, thresholdPx) and returns `DashboardMetrics`.

**Helper functions to create (all internal, not exported):**

1. `classifyRecords(records, thresholdPx)` — returns records annotated with `isSlouching: Math.abs(deltaY) > thresholdPx` (only for non-screen-off records with non-null deltaY)

2. `computeActiveRecords(records)` — filters to `!isScreenOff && deltaY !== null`

3. `computeStreaks(records, thresholdPx)` — walks records and identifies consecutive runs of slouching vs good posture, returning arrays of `{ start: number, end: number, type: 'slouch' | 'good' }`

4. `computeHourlyRates(records, thresholdPx)` — groups active records by `new Date(time).getHours()`, computes slouch rate per hour

5. `determineQuality(sampleCount, minReliable, minLimited)` — returns `'reliable'` if sampleCount >= minReliable, `'limited'` if >= minLimited, else `'insufficient'`

**Metric implementations:**

- **METR-01 postureScore**: `100 - slouchRate - (volatilityPenalty * 0.1) - (correctionPenalty * 0.1)`. Clamp to [0, 100]. volatilityPenalty = min(postureVolatility, 50). correctionPenalty = min(avgTimeToCorrect / 60000, 50) (capped at 50 for >50min correction time). Quality: 'reliable' if activeRecords >= 60, 'limited' if >= 10, 'insufficient' otherwise.

- **METR-02 slouchRate**: `(slouchingRecords.length / activeRecords.length) * 100`. Quality: same thresholds as postureScore.

- **METR-03 avgTimeToCorrect**: For each slouch streak, find the time from streak start to streak end. Average those durations. If no slouch streaks, value is 0. Quality: 'reliable' if >= 3 slouch events, 'limited' if >= 1, 'insufficient' if 0.

- **METR-04 totalScreenTime**: Sum intervals between consecutive non-screen-off records. For each pair of adjacent active records, add `records[i+1].time - records[i].time`. Quality: 'reliable' if active records >= 10, 'limited' if >= 2, 'insufficient' otherwise.

- **METR-05 sessionCount**: `new Set(activeRecords.map(r => r.sessionIndex)).size`. Quality: always 'reliable' (trivially computed).

- **METR-06 longestSlouchStreak**: Max duration (ms) from streaks where type === 'slouch'. Quality: 'reliable' if slouch streaks >= 3, 'limited' if >= 1, 'insufficient' if 0.

- **METR-07 longestGoodStreak**: Max duration (ms) from streaks where type === 'good'. Quality: same logic as longestSlouchStreak.

- **METR-08 breakFrequency**: Identify screen-off gaps (consecutive screen-off records). Count gaps, compute total active time, divide: `totalScreenTime / gapCount`. If no gaps, value is totalScreenTime (one long session). Quality: 'reliable' if gaps >= 3, 'limited' if >= 1, 'insufficient' if 0.

- **METR-09 worstHour**: Hour (0-23) with highest slouch rate from hourly rates. Quality: 'reliable' if >= 3 hours have data, 'limited' if >= 1, 'insufficient' if 0 hours.

- **METR-10 bestHour**: Hour (0-23) with lowest slouch rate from hourly rates (considering only hours with >= 5 active records to avoid noise). Quality: same as worstHour.

- **METR-11 dailyTrend**: Group records by day (Date.getUTCDate). Compute posture score per day. If only 1 day, value is 'insufficient'. If 2+ days: compare first-half average to second-half average. Difference > 5 points = 'improving'/'declining', else 'stable'. Quality: 'reliable' if >= 3 days, 'limited' if 2 days, 'insufficient' if 1 day.

- **METR-12 improvementRate**: Linear regression slope of hourly posture scores over time. X = hours since start, Y = posture score for that hour. Value in score-units per hour. Quality: 'reliable' if >= 4 hourly data points, 'limited' if >= 2, 'insufficient' if < 2.

- **METR-13 severityDistribution**: For each slouching record, classify by `Math.abs(deltaY)`: mild = (threshold, 1.5*threshold], moderate = (1.5*threshold, 2.5*threshold], severe = >2.5*threshold. Quality: 'reliable' if slouching records >= 10, 'limited' if >= 1, 'insufficient' if 0.

- **METR-14 avgTimeToFirstSlouch**: For each session (by sessionIndex), find the first active record and the first slouching record. Compute difference. Average across sessions. If a session has no slouching, exclude it. Value is 0 if no sessions have slouching. Quality: 'reliable' if >= 3 sessions with slouching, 'limited' if >= 1, 'insufficient' if 0.

- **METR-15 postureVolatility**: Standard deviation of all non-null deltaY values. `stddev = sqrt(sum((x - mean)^2) / n)`. Quality: 'reliable' if >= 30 values, 'limited' if >= 5, 'insufficient' if < 5.

- **METR-16 cumulativeSlouchTime**: Sum of durations of all slouch streaks (ms). Quality: same as slouchRate.

- **METR-17 recoverySpeedTrend**: Compare avg time-to-correct in first half of sessions vs second half. If improving (second half faster), 'improving'. If slower, 'declining'. Within 10% difference = 'stable'. Need >= 4 slouch events across >= 2 sessions for 'reliable'. Quality: 'reliable' if >= 4 slouch events in 2+ session halves, 'limited' if >= 2 events, 'insufficient' otherwise.

- **METR-18 slouchByHour**: Array of 24 HourlySlouchRate entries (one per hour 0-23). Hours with no data get slouchRate=0 and sampleCount=0. Quality: 'reliable' if >= 3 hours have data, 'limited' if >= 1, 'insufficient' if 0.

**Edge case handling (per D-12, D-13):**
- Empty active records: return all metrics with quality='insufficient' and sensible default values (0 for numbers, 'insufficient' for trends, empty buckets for distributions).
- Single record: compute what's possible (totalScreenTime=0, sessionCount=1, slouchRate based on that one record, streaks of length 0, etc.).
- Guard every division with `denominator === 0` check, return 0 instead of NaN/Infinity.
