<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL hash -> app state | User-controlled URL hash fragment parsed into view state (range, threshold, engine) |
| Clipboard API | Writing to system clipboard from user action |
| DOM capture -> file download | Capturing DOM content as PNG and triggering browser download |
| CSV content -> file download | Generating CSV from in-memory data and triggering browser download |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | useURLState parseHash | mitigate | Validate all hash params: range values must be positive integers, threshold value must be positive number, engine must be exactly 'recharts' or 'visx'. Malformed values fall back to safe defaults. Never eval() or innerHTML from hash. |
| T-05-02 | Injection | generateRecordsCSV | mitigate | CSV values from PostureRecord are all numbers/booleans/null — no user-supplied strings. Timestamp is Date.toISOString() output (safe). No formula injection risk since no cells start with =, +, -, @. |
| T-05-03 | Information Disclosure | useExportPNG toPng | accept | PNG captures whatever DOM is visible. Since app is client-only with user's own data, no cross-user data risk. User intentionally exports their own dashboard. |
| T-05-04 | Denial of Service | useExportPNG | mitigate | PNG capture on very large DOM could freeze UI. Mitigate with isExporting state to prevent double-clicks and async capture that doesn't block main thread. html-to-image uses canvas API which has browser-imposed size limits. |
| T-05-05 | Tampering | URL hash injection | mitigate | Hash values never inserted into DOM as HTML. All values parsed as numbers or matched against string enum. No XSS vector. |
</threat_model>
