# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Data Pipeline.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Scope Note

Phase 1 delivers one screen: the **upload/landing page**. The visual surface is intentionally minimal — drop zone, file picker trigger, URL-parameter auto-load path, processing state, and error display. No chart, no dashboard. This contract establishes the design tokens, system-preference theming foundation, and copywriting that all subsequent phases inherit.

## Color

Phase 1 establishes the **semantic color foundation** using CSS custom properties. No hard hex values in component code — all colors reference CSS variables that resolve correctly for both light and dark system preference.

Tailwind v4 dark mode strategy: `dark:` variant with `@media (prefers-color-scheme: dark)` (system preference, not class toggle — class toggle comes in Phase 5 with keyboard shortcut `t`).

| Role | Light value | Dark value | Usage |
|------|------------|-----------|-------|
| Dominant (60%) | `oklch(98% 0 0)` (near-white) | `oklch(12% 0 0)` (near-black) | Page background, main surface |
| Secondary (30%) | `oklch(94% 0 0)` (light gray) | `oklch(18% 0 0)` (dark gray) | Drop zone background, card surface |
| Accent (10%) | `oklch(55% 0.2 250)` (blue) | `oklch(65% 0.18 250)` (lighter blue) | Reserved elements listed below |
| Destructive | `oklch(55% 0.22 25)` (red) | `oklch(65% 0.2 25)` (lighter red) | Error state heading, error border |
| Border | `oklch(85% 0 0)` (light border) | `oklch(28% 0 0)` (dark border) | Drop zone border, card border |
| Text primary | `oklch(15% 0 0)` | `oklch(92% 0 0)` | Headings, body copy |
| Text secondary | `oklch(45% 0 0)` | `oklch(60% 0 0)` | Hint text, labels, secondary copy |

**Accent reserved for:**
- The "Choose file" button background (primary action)
- Drop zone border color when in active drag-over state
- Processing spinner/progress indicator
- Success confirmation icon

Accent is NOT used for: body text, headings, borders in default state, background surfaces, icons in idle state.

**Destructive reserved for:**
- Error state card border
- Error heading text
- Error icon

## Interaction States

All interactive elements must express all four states:

| Element | Default | Hover | Active/Pressed | Focus |
|---------|---------|-------|----------------|-------|
| Drop zone | Secondary bg, border dashed | Accent border, slight bg tint | Scale 0.99 | 2px accent outline, 2px offset |
| "Choose file" button | Accent bg, white text | 10% darker accent bg | Scale 0.98 | 2px white outline inside button |
| Drop zone drag-over | Accent border (2px solid), accent bg at 8% opacity | — | — | — |
| Error dismiss (if added) | — | — | — | 2px accent outline |

Transition duration: 150ms ease for all state transitions.

## Accessibility Contract

- Drop zone must have `role="region"` and `aria-label="File drop zone"`
- Drop zone must be keyboard-reachable (tabIndex=0) and respond to Enter/Space to open file picker
- File picker `<input>` must have a visible associated `<label>` (not aria-label only)
- Processing state must announce via `aria-live="polite"` region
- Error messages must be in an `aria-live="assertive"` region
- Color contrast: all text on surface meets WCAG AA (4.5:1 for body, 3:1 for large text/UI components)
- Focus ring: 2px solid accent color, 2px offset — never `outline: none` without replacement

## Token Summary (Tailwind v4 CSS)

These CSS custom properties go in the global CSS entry point (`src/index.css` or equivalent):

```css
@import "tailwindcss";

:root {
  --color-bg: oklch(98% 0 0);
  --color-surface: oklch(94% 0 0);
  --color-accent: oklch(55% 0.2 250);
  --color-destructive: oklch(55% 0.22 25);
  --color-border: oklch(85% 0 0);
  --color-text-primary: oklch(15% 0 0);
  --color-text-secondary: oklch(45% 0 0);
}

@media (prefers-color-scheme: dark) {
  :root {
    --color-bg: oklch(12% 0 0);
    --color-surface: oklch(18% 0 0);
    --color-accent: oklch(65% 0.18 250);
    --color-destructive: oklch(65% 0.2 25);
    --color-border: oklch(28% 0 0);
    --color-text-primary: oklch(92% 0 0);
    --color-text-secondary: oklch(60% 0 0);
  }
}
```

Components use these via Tailwind `[--color-*]` arbitrary values or direct `style` props. No hex in component code.
