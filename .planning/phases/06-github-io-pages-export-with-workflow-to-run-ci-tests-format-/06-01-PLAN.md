**vite.config.ts** -- Add `resolvePagesBase()` function above the `defineConfig` call, and set `base: resolvePagesBase()` in the config object. Per D-10 and D-11:

```typescript
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import tailwindcss from '@tailwindcss/vite';

function resolvePagesBase(): string {
  const repositorySlug = process.env.GITHUB_REPOSITORY?.split('/')[1];

  // Local development or non-GitHub CI: use root
  if (!repositorySlug || process.env.GITHUB_ACTIONS !== 'true') {
    return '/';
  }

  // User/org .github.io repos deploy to root
  if (repositorySlug.endsWith('.github.io')) {
    return '/';
  }

  // Project repos deploy to /<repo-name>/
  return `/${repositorySlug}/`;
}

export default defineConfig({
  base: resolvePagesBase(),
  plugins: [react(), tailwindcss()],
});
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner -> repo | Workflow has read access to repo contents; write access only for Pages |
| CI build -> deployment | Build artifacts uploaded only on main branch; deploy requires OIDC token |
| User browser -> GitHub Pages | Static files served over HTTPS; no server-side processing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | CI workflow | mitigate | Pin all GitHub Actions to major version tags (@v6, @v5, @v4); top-level permissions restricted to contents:read; deploy job has minimal permissions scoped to pages+id-token |
| T-06-02 | Elevation of Privilege | deploy job | mitigate | Deploy gated by if-condition on refs/heads/main; pages:write and id-token:write only on deploy job, not top-level; concurrency group prevents parallel deploys |
| T-06-03 | Denial of Service | CI resources | mitigate | Concurrency groups cancel in-progress runs on same branch; prevents resource exhaustion from rapid pushes |
| T-06-04 | Information Disclosure | build artifacts | accept | Build output is the public-facing app; no secrets in client-side bundle (no env vars injected per D-12); app is client-only with no API keys |
| T-06-05 | Spoofing | npm dependencies | mitigate | npm ci uses lockfile for deterministic installs; no --force or --legacy-peer-deps flags |
| T-06-06 | Repudiation | deployment history | accept | GitHub Pages environments provide deployment history and URLs automatically via deploy-pages action |
</threat_model>
